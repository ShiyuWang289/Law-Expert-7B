# SFT 数据清洗规则决策文档（P2）

> 目标：让每条清洗规则具备“可解释 + 可审计 + 可复盘”能力。  
> 四元组定义：**规则 | 阈值来源 | 命中比例 | 潜在偏置风险**。  
> 本文基于 `advanced_clean.py` 实际运行结果填写（输入：`data/law_qa_alpaca.json`）。

## 1. 规则决策四元组（基于当前运行结果）

### R1. 空值过滤（input/output 为空）
- **规则**：`if not inp or not out: continue`
- **阈值来源**：训练可用样本的最小条件（空字段无监督信号）
- **命中比例**：本轮未命中（基础清洗前后均为 483）
- **潜在偏置风险**：低；仅在上游字段映射异常时可能误删有效样本

---

### R2. 基于 input 的去重
- **规则**：`if inp in duplicate_inputs: continue`
- **阈值来源**：精确去重，减少重复梯度与记忆性过拟合
- **命中比例**：本轮未命中（基础清洗前后均为 483）
- **潜在偏置风险**：可能压缩“同问多答”的答案多样性

---

### R3. 输入过短过滤
- **规则**：`if len(inp) < 5: reject(input_too_short)`
- **阈值来源**：经验阈值；过短问题通常缺乏可判定上下文
- **命中比例**：0 / 483 = **0.00%**
- **潜在偏置风险**：可能误伤极短但有效咨询句

---

### R4. 输出过短过滤
- **规则**：`if len(out) < 30: reject(output_too_short)`
- **阈值来源**：法律问答需“结论 + 依据/解释”，短答监督信息弱
- **命中比例**：0 / 483 = **0.00%**
- **潜在偏置风险**：可能误删“短但正确”的结论型回答

---

### R5. 总长度上限过滤
- **规则**：`if len(inp) + len(out) > 1500: reject(total_too_long)`
- **阈值来源**：与训练 `cutoff_len` 工程约束对齐，降低截断风险
- **命中比例**：0 / 483 = **0.00%**
- **潜在偏置风险**：可能误删复杂案情长答案，削弱长上下文适应能力

---

### R6. URL 噪声过滤
- **规则**：`if re.search(r'https?://|www\.', out): reject(contains_url)`
- **阈值来源**：链接常对应搬运/广告/无关噪声，影响法律回答纯净度
- **命中比例**：0 / 483 = **0.00%**
- **潜在偏置风险**：可能误删“包含权威来源链接”的高质量答案

---

### R7. 特殊字符比例过滤
- **规则**：`if special_char_ratio > 0.15: reject(too_many_special_chars)`
- **阈值来源**：启发式可读性红线；异常符号堆积常伴随乱码
- **命中比例**：0 / 483 = **0.00%**
- **潜在偏置风险**：可能误伤合法格式化文本（条款编号、列表、引文）

---

### R8. 鹦鹉式回复过滤
- **规则**：`if out.startswith(inp[:20]): reject(parrot_response)`
- **阈值来源**：任务目标是“信息增量”，非复述问题
- **命中比例**：0 / 483 = **0.00%**
- **潜在偏置风险**：可能误删“先复述再回答”的规范话术

---

### R9. 法律相关性关键词过滤（本轮唯一命中）
- **规则**：`if not any(kw in out for kw in legal_keywords): reject(low_legal_relevance)`
- **阈值来源**：领域一致性兜底；关键词作为低成本弱监督判别信号
- **命中比例**：2 / 483 = **0.41%**
- **总体删除贡献**：2 / 483 = **0.41%**
- **潜在偏置风险**：
  - 可能误删语义上合法律、但未显式包含关键词的样本；
  - 强化“关键词显式表达”偏好，削弱表达多样性。

## 2. 当前版本结论

1. 本轮数据质量整体较高：基础清洗阶段与大部分质量规则均未触发删除；  
2. 质量阶段仅删除 2 条（0.41%），说明当前规则更偏“兜底保障”而非“大幅裁剪”；  
3. 唯一命中的法律相关性规则有效提升领域一致性，但需关注关键词法带来的潜在误伤。

- 对 `low_legal_relevance` 删除样本做人工抽检（建议全量抽检，当前仅 2 条）；  
- 若发现误删，考虑引入“关键词 + 语义分类器”双判定，降低规则偏置；  
- 在 P3 对照实验中验证：该规则是否对下游表现（法条准确率/关键点覆盖率）有正收益。
